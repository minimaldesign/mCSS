## The problem

Styling a native `<select>` dropdown indicator is one of those things that _should_ be simple but never is. You want a custom chevron icon that:

1. Matches the current **text color** (so it works in both light and dark themes).
2. Doesn't require a **wrapper element** around the `<select>`.
3. Uses a **single asset** (no separate light/dark SVGs).

The obvious solution — `background-image` with an inline SVG — fails requirement #1. Inside a `url()` data URI, `currentColor` doesn't resolve against the page's `color` property. The SVG is a separate image document. Your CSS custom properties and `light-dark()` functions don't exist there.

The _other_ obvious solution — a single `mask-image` of the chevron with a colored background — fails because `mask` applies to **everything** the element paints: background, border, _and the selected option text_. You end up with invisible text shaped like a chevron.

## The idea: three mask layers

The trick is to use **three** mask layers composited together so that different parts of the element are selectively revealed:

1. **Layer 3 (bottom):** A solid opaque fill covering the entire **border-box**. This is the base — it keeps the border and everything else visible.
2. **Layer 2 (middle):** A gradient that is transparent on the left and opaque on the right (where the chevron goes), clipped to **padding-box**. Composited with **`exclude`** (XOR), this punches a hole in the right strip of the padding area — but _can't touch the border_ because it's clipped to a smaller box.
3. **Layer 1 (top):** The chevron SVG, also clipped to **padding-box**. Composited with **`add`**, it fills back in _only_ the chevron strokes within the punched-out strip.

The result:

- **Left side (text):** Opaque. The base layer is untouched — the selected option text renders normally.
- **Right strip (chevron strokes):** Opaque. The chevron `add`s back what `exclude` removed.
- **Right strip (non-chevron):** Transparent. Shows the page background through.
- **Border:** Opaque on all four sides. Layer 2 can't reach it.

Then, a simple **background gradient** does the coloring: `--body-background-color` on the left, `--text-color` on the right. The mask controls _where_ each color is visible. The chevron strokes reveal the text-color band. Since `--text-color` is a theme token (typically set with `light-dark()`), the chevron automatically adapts to the current theme.

## The implementation

```css
select {
  --icon-size: 24px;
  --split: calc(100% - var(--icon-size));

  appearance: none;
  border: 1px solid var(--theme-border-color);
  border-radius: var(--input-border-radius);
  padding: var(--xs1) calc(var(--input-padding) + var(--md1)) var(--xs1)
    var(--input-padding);

  /* Left = field background, right = text color (for the chevron) */
  background: linear-gradient(
    to right,
    var(--body-background-color) 0 var(--split),
    var(--text-color) var(--split) 100%
  );

  /* Three mask layers: chevron, punch-out strip, solid base */
  mask-image:
    url("../assets/icons/chevron-down.svg"),
    linear-gradient(to right, transparent var(--split), red 0),
    linear-gradient(red 0 0);
  mask-size: var(--icon-size), auto, auto;
  mask-position:
    calc(100% - var(--xs1)) center,
    0 0,
    0 0;
  mask-repeat: no-repeat;
  mask-clip: padding-box, padding-box, border-box;
  mask-origin: padding-box, padding-box, border-box;
  mask-composite: add, exclude;
}
```

Let's break it down.

### The background gradient

```css
background: linear-gradient(
  to right,
  var(--body-background-color) 0 var(--split),
  var(--text-color) var(--split) 100%
);
```

A hard-edge gradient with two bands. The left band matches the field's background. The right band is the text color — this is the "paint" that will show through the chevron mask. Because `--text-color` is a theme-aware token, the chevron color automatically follows the current theme.

### The mask stack

**Layer 3** — the solid base:

```css
linear-gradient(red 0 0)
```

The simplest way to produce a solid opaque image value. `mask-image` doesn't accept a plain color — it needs an `<image>`. A single-stop gradient fills the entire area. The color is irrelevant (masks use the alpha channel); `red` just makes it obvious this is mask code, not a visible color.

Clipped to **`border-box`** so it covers the border too.

**Layer 2** — the punch-out strip:

```css
linear-gradient(to right, transparent var(--split), red 0)
```

Transparent on the left, opaque on the right (same split point as the background). Composited with **`exclude`** (XOR) against the solid base:

- Left: `XOR(0, 1) = 1` — stays opaque.
- Right: `XOR(1, 1) = 0` — punched out.
- Border area: layer 2 is `padding-box` clipped so it's invisible there — `XOR(0, 1) = 1` — border stays.

Why `exclude` instead of `subtract`? `exclude` is commutative (A XOR B = B XOR A), so it produces the correct result regardless of whether the browser evaluates mask compositing as source-minus-destination or destination-minus-source. With `subtract`, some browsers silently flip the operands.

**Layer 1** — the chevron:

```css
url("../assets/icons/chevron-down.svg")
```

The SVG is loaded as an image. In alpha mask mode (the default for SVG images in `mask-image`), the opaque stroke pixels have alpha 1 and everything else is transparent. Composited with **`add`** to fill back the chevron shape within the punched-out strip.

### Positioning the chevron

```css
mask-position:
  calc(100% - var(--xs1)) center,
  0 0,
  0 0;
```

Only the chevron layer needs an offset — the gradient and solid layers fill their entire box, so `0 0` has no visible effect on them.

Using `calc(100% - offset)` instead of the three-value `right <length> center` syntax keeps things robust. Be careful with a single `mask-position` value when layers have different sizes: the percentage math `(container - image) * %` can shift full-size gradient layers unexpectedly.

### The `&[multiple]` reset

```css
&[multiple] {
  background: var(--body-background-color);
  mask-image: none;
}
```

Multi-selects don't need a dropdown indicator. Clear both the gradient and mask to get a plain field.

## Gotchas

**The transparent gap.** The right strip is masked out entirely — non-chevron pixels are fully transparent. Whatever sits _behind_ the `<select>` (typically the page background) shows through. This is invisible as long as `--body-background-color` matches the page background. If the select sits on a card or stripe with a different color, you'll see a faint rectangle.

**`mask-composite` browser support.** Standard `mask-composite` (`add`, `exclude`) is supported in Chrome 120+ and Firefox. Safari needs the `-webkit-mask-composite` prefix with different keyword names (`source-over`, `xor`). If you use autoprefixer / PostCSS, it should handle the mapping. Test in Safari.

**Outlines over borders.** Because the mask can interact with the border in subtle ways, the `:focus-visible` style uses `outline` with a positive `outline-offset` rather than modifying the border.
